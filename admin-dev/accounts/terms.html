If the latency across the BC IP network exceeds one hundred twenty (120) milliseconds, BC will classify this as Downtime, or

(c) A fault in server hardware by which the Services are provided. If time between Customer's formal indication of service fault and BC's acknowledgement and response to begin resolution of the hardware fault exceeds four (4) hours, BC will classify this as Downtime.
